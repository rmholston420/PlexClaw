# PlexClaw docs

This directory collects deeper project documentation that sits behind the main repository landing page.

## Documents

- `competitive-gui-roadmap.md` — product direction and competitive targets for PlexClaw as a Claude Code and Claude Agent SDK browser GUI.
- `frontend-architecture.md` — frontend structure, browser responsibilities, and UI architecture notes.
- `frontend-phase-plan.md` — staged implementation plan for frontend capability expansion.
- `CANNIBALIZATION.md` — notes on overlap, differentiation, and product-boundary concerns.

## Reading order

If you are new to the repo, start here:

1. Main project overview and quickstart in the repository `README.md`
2. Contributor and runtime conventions in `AGENTS.md`
3. Claude-native project memory in `CLAUDE.md`
4. Product and architecture detail in the docs listed above

## Documentation intent

The main `README.md` should stay concise and product-facing. This docs index exists so deeper operator, architecture, and planning material can grow without turning the landing page into a full manual.
